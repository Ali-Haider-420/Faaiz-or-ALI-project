# Supabase Setup Guide

This project uses **Supabase** for Authentication and Database.

## 1. Environment Variables

Create a project in [Supabase](https://supabase.com/).
Get your URL and Anon Key from `Project Settings > API`.

Add these to your `.env.local` file:

```env
NEXT_PUBLIC_SUPABASE_URL=your-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

## 2. Database Schema

Run the following SQL in the Supabase SQL Editor to create the necessary tables.

### Contact Form Submissions

```sql
create table contact_submissions (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  name text,
  email text,
  phone text,
  message text
);

-- Enable Row Level Security (RLS)
alter table contact_submissions enable row level security;

-- Create Policy to allow anonymous inserts (since the contact form is public)
create policy "Enable insert for anyone"
on contact_submissions
for insert
with check (true);

-- Create Policy to allow only authenticated admins to view (Optional)
-- You can adjust this based on your auth roles
create policy "Enable select for authenticated users only"
on contact_submissions
for select
to authenticated
using (true);
```

### Authentication

Supabase Auth is handled automatically. The `src/app/auth/actions.ts` file handles sign up and sign in using `supabase.auth.signUp` and `supabase.auth.signInWithPassword`.

- **Redirect URL**: In Supabase Authentication > URL Configuration, set the "Site URL" to your production URL (or `http://localhost:3000` for dev).
- **Email Confirmations**: By default, Supabase sends confirmation emails. The `src/app/auth/callback/route.ts` file handles the link click.
